# Promr Brand Chart — 디자인용 프롬프트

디자인 툴(Figma, AI UI 생성 등)에 넣을 때 복사해서 쓰면 되는 문장들입니다.

---

## 1. 앱 한 줄 설명 (컨텍스트)

```
법인 실적·처방사진 업로드와 제약사 법인 정산 확인을 위한 B2B 웹 앱. 
법인은 병의원(또는 전체)과 월을 선택한 뒤 업로드한다. 
사용자 역할은 법인(업로드) / 제약사(정산 표 확인) 두 가지.
```

---

## 2. 공통 레이아웃

```
좌측 고정 사이드바(세로 메뉴) + 오른쪽 넓은 메인 영역.
사이드바: 너비 220px, 흰 배경, 세로 나열 메뉴, 하단에 역할 선택(법인/제약사) 드롭다운.
메인: 가로로 넓게(표가 잘 보이도록), 최대 2400px까지 확장 가능.
전체 톤: 밝은 배경(#f8fafc), 카드/테이블은 흰색, Primary 블루(#2563eb), 보조 텍스트 회색(#64748b).
```

---

## 3. 화면별 프롬프트

### 3-1. 홈

```
타이틀: "Promr Brand Chart"
부제: "법인 실적·처방사진 업로드 및 제약사 법인 정산 확인"

카드 그리드(반응형, 최소 260px 카드):
- 법인일 때: 카드 2개
  - "실적 업로드" / "엑셀 파일로 판매 실적을 업로드합니다." / "업로드 페이지로 →"
  - "처방사진 업로드" / "실적에 대한 처방 사진을 병원별 또는 전체로 업로드합니다." / "업로드 페이지로 →"
- 제약사일 때: 카드 1개
  - "법인 정산 확인" / "법인별 실적을 확인하고 정산합니다. 법인·병의원 필터, 품목 검색, 합계 열 제공." / "정산 확인 페이지로 →"
```

### 3-2. 실적 업로드 (법인 전용)

```
페이지 타이틀: "실적 업로드"
설명: "병의원과 월을 선택한 뒤 엑셀 파일을 업로드하여 판매 실적을 등록합니다."

선택 폼 (가로 나열):
- "병의원" 셀렉트 — 옵션: "전체" + 해당 법인 소속 병의원 목록
- "월" 셀렉트 — 옵션: 2024년 01월 ~ 현재까지 등 (최근 24개월)

다운로드 버튼 3개 (나란히):
- "엑셀 양식 다운로드"
- "품목 다운로드"
- "거래처 다운로드"

드래그 앤 드롭 영역:
- 문구: "엑셀 파일을 여기에 놓거나 클릭하여 선택하세요 (.xlsx, .xls)"
- 점선 테두리, 호버 시 Primary 색 강조

미리보기 테이블 (있을 때):
- 헤더: 병의원 | 제품명 | 수량 | 금액
- 행 반복

주요 CTA 버튼: "실적 등록하기" (Primary 블루)
```

### 3-3. 처방사진 업로드 (법인 전용)

```
페이지 타이틀: "처방사진 업로드"
설명: "병의원과 월을 선택한 뒤 실적에 대한 처방 사진을 업로드합니다. 병의원은 전체를 선택할 수 있습니다."

폼 순서 (법인 선택 없음 — 로그인한 법인 기준):
1. "월" — 셀렉트 (최근 24개월 등)
2. "병의원" — 셀렉트 (옵션: "전체" + 해당 법인 소속 병의원 목록)
3. "처방 사진 (해당 실적: N건)" — 이미지 드래그 앤 드롭 영역
   - 문구: "이미지를 여기에 놓거나 클릭하여 선택하세요"
   - 선택한 이미지 썸네일 그리드

버튼: "처방사진 등록"
```

### 3-4. 법인 정산 확인 (제약사 전용)

```
페이지 타이틀: "법인 정산 확인"
설명: "법인별 실적을 확인하고 정산합니다. 법인·병의원 필터와 품목 검색을 사용할 수 있습니다."

필터 행 (가로 나열):
- "법인" 셀렉트 (전체 + 법인 목록)
- "병의원" 셀렉트 (전체 + 병의원 목록)
- "품목 검색" 검색 입력 (placeholder: "제품명으로 검색")

정산 테이블 (가로 스크롤 가능, 2단 헤더):
- 1행: No. | 상태 | 정산월 | 처방월 | 거래처코드 | 거래처명 | 사업자번호 | 주소 | 금액 | [원내 2칸] | [원외 2칸]
- 2행(헤더): (위 9개 컬럼은 rowSpan) | 품목수 ▼ | 처방액 ▼ | 품목수 ▼ | 처방액 ▼
- 데이터 행: 번호, 승인, 2026-02, 2026-01, 코드, 거래처명, 사업자번호, 주소, 금액(소수2자리), 원내 품목수 0, 원내 처방액 0.00, 원외 품목수, 원외 처방액
- 맨 아래 합계 행: "합계" + 각 컬럼 합계, 굵게 구분선 위

금액·원내/원외 숫자는 우측 정렬. 헤더에 정렬 아이콘(▼) 표시.
```

## 4. 사이드 메뉴 (라벨만)

```
법인일 때:
- 홈
- 실적 업로드
- 처방사진 업로드

제약사일 때:
- 홈
- 법인 정산 확인

사이드바 맨 아래: 드롭다운 "법인" / "제약사"
현재 페이지 메뉴는 Primary 색 + 연한 배경으로 강조.
```

## 6. 짧은 프롬프트 (AI 이미지/UI 생성용)

```
B2B 웹 앱 대시보드. 왼쪽 220px 세로 사이드 메뉴(홈, 실적 업로드, 처방사진 업로드, 법인 정산 확인), 오른쪽 넓은 메인. 밝은 회색 배경, 흰색 카드/테이블, 블루 액센트. 법인·제약사 정산·업로드용.
```

```
정산 화면: 상단에 법인/병의원/품목 검색 필터, 아래 넓은 데이터 테이블. 2단 헤더(원내/원외 각각 품목수·처방액), No·상태·정산월·처방월·거래처코드·거래처명·사업자번호·주소·금액 컬럼. 하단 합계 행. 미니멀 라이트 테마.
```

```
실적 업로드 화면: 타이틀 "실적 업로드", 엑셀 양식/품목/거래처 다운로드 버튼 3개, 큰 드래그앤드롭 영역("엑셀 파일을 여기에 놓거나 클릭"), 미리보기 테이블(병의원·제품명·수량·금액), "실적 등록하기" 버튼.
```
